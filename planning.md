# FitFindr — planning.md

## Short Complete Interaction Description
FitFindr is a secondhand fashion assistant that searches for listings matching user queries, recommends outfit combinations using the user's existing wardrobe, and generates shareable social media fit cards. The search_listings tool is triggered first by the parsed query parameters; if successful, it selects the best match, which triggers suggest_outfit to pair the item with the wardrobe, and finally create_fit_card is called to generate a caption. If search finds nothing, the loop sets an error and halts early, and if suggest_outfit fails, it halts or falls back to general styling advice without calling subsequent tools.

---

## Tools

### Tool 1: search_listings

**What it does:**
Searches a mock database of secondhand clothing listings for items matching the user's keyword description, size, and maximum price.

**Input parameters:**
- `description` (str): Search keyword(s) describing the item (e.g. "vintage graphic tee").
- `size` (str | None): Optional size filter (e.g. "M", "S/M"). Case-insensitive match.
- `max_price` (float | None): Optional maximum price filter (inclusive).

**What it returns:**
- `list[dict]`: A list of matching listing dictionaries (sorted by relevance score), where each dict contains fields: `id`, `title`, `description`, `category`, `style_tags`, `size`, `condition`, `price`, `colors`, `brand`, and `platform`.

**What happens if it fails or returns nothing:**
Returns an empty list `[]` instead of crashing. The planning loop detects this empty list, updates the session with a helpful error message, and stops execution early.

---

### Tool 2: suggest_outfit

**What it does:**
Generates complete outfit suggestions pairing the selected listing with items from the user's wardrobe using the Groq LLM.

**Input parameters:**
- `new_item` (dict): The listing dictionary representing the thrifted clothing item to buy.
- `wardrobe` (dict): The user's wardrobe dictionary containing a list of item dictionaries under the `items` key.

**What it returns:**
- `str`: A detailed text description of 1-2 outfit suggestions incorporating the new item.

**What happens if it fails or returns nothing:**
If the wardrobe is empty or minimal, it returns useful general styling advice for the item instead of crashing. If the LLM API call fails, it catches the exception and returns a helpful error string.

---

### Tool 3: create_fit_card

**What it does:**
Generates a short, creative, and shareable social media OOTD caption (Instagram/TikTok style) showcasing the new outfit.

**Input parameters:**
- `outfit` (str): The outfit suggestion string generated by suggest_outfit.
- `new_item` (dict): The listing dictionary of the thrifted item.

**What it returns:**
- `str`: A 2-4 sentence caption that casually mentions the item name, price, and platform naturally, capturing the vibe of the outfit.

**What happens if it fails or returns nothing:**
If the outfit input is empty, invalid, or the API call fails, it returns a descriptive error message string instead of crashing.

## Planning Loop

**How does your agent decide which tool to call next?**
The planning loop operates sequentially with early-termination branching:
1. Start with user query and extract/parse query parameters: `description`, `size`, `max_price`.
2. Call `search_listings(description, size, max_price)`.
3. If search results are empty:
   - Set `session["error"] = "No listings found matching your description. Try loosening your size, price, or description filters."`.
   - Do not call `suggest_outfit` or `create_fit_card`.
   - Return `session` early.
4. If search results exist:
   - Set `session["search_results"] = results`.
   - Set `session["selected_item"] = results[0]` (highest scoring listing).
   - Call `suggest_outfit(session["selected_item"], wardrobe)`.
   - Set `session["outfit_suggestion"] = outfit_text`.
   - If `outfit_suggestion` is missing or empty, set `session["error"] = "Failed to suggest outfit."` and return early.
   - Call `create_fit_card(session["outfit_suggestion"], session["selected_item"])`.
   - Set `session["fit_card"] = fit_card_text`.
   - Return `session`.

## State Management

**How does information from one tool get passed to the next?**
Information is managed and passed via a mutable `session` dictionary created at the start of the interaction. The session tracks:
- `query` (str): The raw user query.
- `parsed` (dict): Extracted fields `{description: str, size: str | None, max_price: float | None}`.
- `search_results` (list[dict]): Saved outputs from `search_listings`.
- `selected_item` (dict | None): The chosen listing (highest score) passed to `suggest_outfit` and `create_fit_card`.
- `wardrobe` (dict): The input user wardrobe.
- `outfit_suggestion` (str | None): Saved output from `suggest_outfit`, passed to `create_fit_card`.
- `fit_card` (str | None): Final output from `create_fit_card` shown to the user.
- `error` (str | None): Explains why execution stopped early (if any).

---

## Error Handling

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| `search_listings` | No results match the query | Set `session["error"]` to a helpful message and stop early (do not call subsequent tools). |
| `suggest_outfit` | Wardrobe is empty | Fall back to general styling advice for the item and proceed. |
| `create_fit_card` | Outfit input is missing or incomplete | Return a descriptive error message indicating missing outfit details instead of crashing. |
| Groq API | API key missing, rate limits, or network failure | Catch exception in tools, return a user-friendly error message, and set `session["error"]` to terminate. |
| User Input | Malformed or missing search criteria | Fall back to default description search or prompt for clarification. |

```mermaid
    UserQuery([User Query]) --> Parsing[Parse Query: Description, Size, Max Price]

    InitSession --> CallSearch[Call search_listings]
    CallSearch --> CheckResults{Are results empty?}
    CheckResults -- Yes --> SetError[Set session error message]
    SetError --> StopEarly[Stop & Return Session]
    CheckResults -- No --> SetSelected[Set selected_item = results[0]]
    SetSelected --> CallSuggest[Call suggest_outfit]
    CallSuggest --> CheckOutfit{Is outfit empty/error?}
    CheckOutfit -- Yes --> SetOutfitError[Set session error message]
    SetOutfitError --> StopEarly2[Stop & Return Session]
    CheckOutfit -- No --> SetOutfit[Set session outfit_suggestion]
    SetOutfit --> CallFitCard[Call create_fit_card]
    CallFitCard --> SetFitCard[Set session fit_card]
    SetFitCard --> ReturnSession([Return Session State])
```

## A Complete Interaction (Step by Step)

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:**
The planning loop parses the query to extract parameters:
- `description`: "vintage graphic tee"
- `size`: `None` (not mentioned in query)
- `max_price`: `30.0`

It calls `search_listings("vintage graphic tee", size=None, max_price=30.0)`. The tool scores and filters mock listings, returning `lst_006` ("Graphic Tee — 2003 Tour Bootleg Style") and `lst_033` ("Vintage Band Tee") as matches.

**Step 2:**
The planning loop selects the top result `lst_006` and stores it in `session["selected_item"]`.
It calls `suggest_outfit(session["selected_item"], wardrobe)` passing the user's wardrobe (containing baggy jeans and chunky sneakers).
The LLM returns an outfit suggestion string recommending pairing the bootleg graphic tee with the wardrobe's baggy straight-leg jeans, black crossbody bag, and chunky white sneakers. This is stored in `session["outfit_suggestion"]`.

**Step 3:**
The planning loop calls `create_fit_card(session["outfit_suggestion"], session["selected_item"])`.
The LLM generates a casual, OOTD social media style caption: *"Nostalgic vibes today in this 2003 Tour Bootleg style graphic tee from depop for only $24! Paired it with my go-to baggy dark wash jeans and chunky white sneakers. Perfect streetwear look. #OOTD #thriftfinds"*
This is stored in `session["fit_card"]`.

**Final output to user:**
The Gradio UI maps the session outputs, presenting the top listing details, the custom outfit recommendation, and the final social media fit card caption.
